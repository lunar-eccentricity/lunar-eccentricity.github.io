0.1 位、字节、字的关系

字符串都被储存在char类型的数组中，数组由连续的存储单元组成，字符串中的字符被储存在相邻的存储单元中，每个单元储存一个字符。数组末尾的字符\0，是空字符、非打印字符，用它标记字符串的结束。C中的字符串一定以空字符结束，这意味着数组的容量至少比待存储字符串中的字符数多1.

数组是同类型数据元素的有序序列

2. define指令

用预处理器来定义常量

4.2 表达式和语句

1. 表达式

表达式由运算符和运算对象组成
运算对象可以是常量、变量或者二者的结合
每个表达式都有一个值

2. 语句

语句是C程序的基本构建块。一条语句相当于一条完整的计算机指令，大部分语句以分号结尾
最简单的语句是空语句
语句可以改变值和调用函数
声明不是表达式语句，如果删除声明后面的分号，剩下的部分不是一个表达式，也没有值

while语句有三个不同的部分。
关键字while
圆括号中是待测试条件
如果测试条件为真，执行循环体中的语句
如果while循环中只有一条语句，不需要花括号括起来

副作用是对数据对象或文件的修改

它的副作用是将变量的值设置为50
从C语言的角度看，主要目的是对表达式求值，其它就是副作用

序列点是程序执行的点，在该点上，所有的副作用都在进入下一步之前发生
在C语言中，语句中的分号标记了一个序列点。在一个语句中，赋值运算符、递增运算符和递减运算符对运算对象做的改变必须在程序执行下一条语句之前完成
任何一个完整表达式的结束也是一个序列点

完成表达式指这个表达式不是另一个更大表达式的子表达式
序列点有助于分析后缀递增何时发生

3. 复合语句（块）

复合语句是用花括号括起来的一条或多条语句，也称为块
缩进对编译器不起作用，编译器通过花括号和while循环的结构来识别和解释指令
使用缩进可以为读者指明程序的结构

4. 类型转换

在语句和表达式中应使用类型相同的变量或者常量
C可以采用一套规则进行自动类型转换，但是有一点的危险性

    如果scanf()在转换值之前出了问题（例如，检测到文件结尾或遇到硬件问题）会返回一个特殊值EOF（其值通常被定义为-1），这个值也会引起循环终止

告知循环何时停止：

3. for循环

for循环把初始化、测试和更新组合在一起

关键字for后面有三个表达式，分别用两个分号隔开，括号外不用分号

  1. 初始化，只会在for循环开始时执行一次（赋初值）
  2. 测试条件，在执行循环之前对表达式求值。如果表达式为假，循环结束（列范围）
  3. 执行更新，在每次结束时求值（递增计数器）

4. 其他赋值运算符

5. 逗号运算符

  • 保证了被它分割的表达式从左往右求值
    • 逗号是一个序列点，所以逗号左侧项的所有副作用都在程序执行逗号右侧项之前发生
  • 整个逗号表达式的值是右侧项的值

6. 出口条件循环 do while

  • 入口条件循环：在循环的每次迭代之前检查测试条件，所以有可能根本不执行循环体中的内容
  • 出口条件循环：在循环的每次迭代之后检查测试条件，这保证了至少执行一次循环体中的内容

do while循环的通用形式

do while循环在执行完循环体后才执行测试条件，所以至少执行循环体一次；而for循环或者while循环都是在执行循环体之前先执行测试条件。do while循环适用于那些至少要迭代一次的循环。

7. 如何选择循环

入口循环用得比较多

  • 一般原则是在执行循环之前测试条件比较好
  • 测试放在循环的开头，程序的可读性更高
  • 当循环涉及到初始化和更新变量时，用for循环比较合适，其他情况用while循环

9.1.2 分析程序

void starbar(void)；

  • 圆括号表明 starbar 是一个函数名
    • 第一个 void 是函数类型，void 类型表明函数没有返回值
    • 第二个 void 表明该函数不带参数
  • 分号表明是在声明函数，不是在定义函数
  • 一般而言，函数原型指明了函数的返回值类型和函数接受的参数类型，这些信息成为该函数的签名
  • 对于starbar（）函数而言，其签名是该函数没有返回值，没有参数
  • 程序把starbar()原型置于main()的前面，也可以放在main()内部的声明变量处
  • 程序中starbar()和main()的定义形式相同
    • 函数头包括函数类型、函数名和圆括号
    • 接着是花括号内部的变量声明、函数表达语句
    • 注意函数头不加分号表明是在定义函数
  • 程序可以把starbar()和main()放在一个文件中，也可以把它们分别放在两个文件中
    • 把函数放在一个文件中的单文件形式比较容易编译，而使用多个文件方便在不同的程序中使用同一个函数
    • 如果把函数放在一个单独的文件中，要把#define 和#include 指令也放入该文件

注意

  • 对于不识别 ANSI C 风格原型的编译器，只需声明函数的类型 void starbar()
  • 一些老版本的编译器连 void 都识别不了，需要把没有返回值的函数声明为int 类型
  • starbar()函数中的变量count是局部变量，变量只属于starbar()函数，可以在程序中的其他地方使用count变量名
  • starbar()不需要与主函数通信

9.1.8 使用 return 从函数中返回值

前面介绍如何把信息从主调函数传递给被调函数，反过来，函数的返回值可以把信息从被调函数传回主调函数

  • imin() 函数被设计用来处理 int 类型的值，返回值为int，因此函数类型为 int

  • 同时创建了一个简单的 main() ,用于检查 imin() 是否正常工作；这种被设计用于测试函数的程序有时被称为驱动程序，该驱动程序调用一个函数

  • 关键字 return 后面的表达式的值就是函数的返回值

  • 变量 min 属于 imin() 函数私有，但是 return 语句把 min 的值传回来主调函数，但并不是把变量 min 传回主调函数

    • lesser = imin(n,m); √
    • imin(n,m); lesser = min; ×
    • 主调函数并不知道变量 min 的存在，imin() 中的变量是 imin() 的局部变量，函数调用 imin(evil1, evil2) 只是把两个变量的值拷贝了一份

  • 返回值不仅可以返回变量的值，还可以返回任意表达式的值

    • return (n, m) ? n : m;
    • 虽然这里不要求用圆括号把返回值括起来，但是如果想让程序条理更加清楚或统一风格，可以把返回值放在圆括号内

  • 如果函数返回值与函数声明的类型不匹配，实际得到的返回值相当于把函数中指定的返回值赋给与函数类型相同的变量所得到的值

  • 使用 return 语句的另一个作用是终止函数并把控制返回给主调函数的下一条语句。因此还可以这样编写 imin():

  • 在函数中使用多个 return 也没有错

  • return语句执行后会立即返回主调函数，后面的语句不会被执行

  • 可以使用 return; 导致终止函数，并把控制返回给主调函数，由于 return 后面没有任何表达式，所以没有返回值，只有在 void 类型函数中才会用到这种形式

9.1.9 函数类型

  • 声明函数必须声明函数类型
    • 带返回值的函数类型应与其返回值类型相同
    • 没有返回值的函数应声明为 void 类型
    • 如果没有声明函数类型，旧版 c 编译器回家顶函数类型是 int ，C99标准不再支持 int 类型函数的假定设置
  • 类型声明是函数定义的一部分，函数类型之恶是返回值的类型，不是函数参数的类型
  • 要正确地使用函数，程序在第一次使用函数之前必须知道函数的类型，方法之一是把完整的函数定义放在第一次调用函数的前面，通常地做法是提前声明函数，把函数信息告知编译器
  • int imin(int, int); 说明 imin 是一个函数名，有两个 int 类型的形参，且返回int类型的值
  • 可以把函数的前置声明放在主调函数外面，也可以放在主调函数里面，但函数原型都声明在使用函数之前
  • ANSI C 标准库中，函数被分成多个系列，每个系列都有各自的头文件，这些头文件除了其他内容，还包含本系列所有函数的声明
  • math.h 头文件包含了各种数学函数的声明

9.2 ASCI C 函数原型

在 ASCI C 标准前，因为只需要声明函数的类型，不用声明任何参数，如果调用函数时使用的参数个数不对或类型不匹配，编译器根本不会察觉出来

9.2.1 问题所在

  • 主调函数把它的参数存储在名为栈的临时存储区，被调函数从栈中读取这些参数
  • 如果调用函数时参数类型不匹配，则这个过程并未相互协调
  • 主调函数根据函数调用中的实际参数决定传递的类型，而被调函数根据它的形式参数读取值
  • 函数调用 imax(2) 把一个整数放在栈中，当 imax() 函数开始执行时，它从栈中读取两个整数，而实际栈中只存放了一个待读取的整数，所以读取的第2个值时当时恰好在栈中的其他值
  • 第二次使用 imax() 函数时，它传递的是 float 类型的值，这次把两个 double 类型的值放在栈中，在系统中，两个 double 类型的值就是两个 64 位的值，所以 128 位的数据被放在栈中。当 imax() 函数从栈中读取两个 int 类型的值时，他从栈中读取前64位。在系统中，int 类型的变量占用 32 位，这些数据对应两个整数

9.2.2 ANSI 的解决方案

针对参数不匹配的问题，ASCI C 标准要求函数声明时要声明变量的类型
使用函数原型来声明函数的返回类型、参数的数量和每个参数的类型
int imax(int, int); ; int imax(int a, int b);
第一种形式以逗号分隔类型列表
第二种在类型后面添加了变量名
这里的变量名是假名，不必与函数定义的形式参数名一致

如果两个参数都是数字，但类型不匹配，编译器会把实际参数的类型转换为形式参数的类型

错误和警告的区别是：错误导致无法编译，警告仍然允许编译

9.3 递归

C 允许函数调用它自己，这种调用过程称为递归
结束递归是使用递归的难点，如果递归代码中没有终止递归的条件测试部分，一个调用自己的函数会无限递归
可以使用循环的地方通常都可以使用递归
递归方案更简洁，但效率没有循环高；两者各有适用条件

1 演示递归

演示四级递归

当 n 变化是，内存会堆叠栈点，cpu会先保护断点，保护现场，离开当前函数，前往上一个栈点，当解决后再解决下一个

递归工作过程：

  1. main() 调用了带参数1的 up_and_down() 函数，执行结果是 up_and_down() 中的形参值为 1，此时打印语句为 Level 1；
  2. 由于 n < 4，执行 up_and_down(n + 1); 语句，导致 n 值变为 2，形参值改变，栈点加一，返回 #1 打印 Level 2
  3. 由于 n < 4，执行 up_and_down(n + 1); 语句，导致 n 值变为 3，形参值改变，栈点加一，返回 #1 打印 Level 3
  4. 由于 n < 4，执行 up_and_down(n + 1); 语句，导致 n 值变为 4，形参值改变，栈点加一，返回 #1 打印 Level 4
  5. 由于 n = 4，执行 #2 语句，打印 LEVEL 4
  6. 此时栈减一，导致 n 值变为 3，继续从之前的断点运行，执行 #2 打印 LEVEL 3
  7. 此时栈减一，导致 n 值变为 2，继续从之前的断点运行，执行 #2 打印 LEVEL 2
  8. 此时栈减一，导致 n 值变为 1，继续从之前的断点运行，执行 #2 打印 LEVEL 1
  9. 返回主调函数
  10. Level 1 与 LEVEL 1 的地址相同，level 1 与 level 2 的地址不同

9.3.2 递归的基本原理

  1. 每级函数调用都有自己的变量，第1级的n与第4级的n知识变量名相同，但是值不相同，当程序最终返回 up_and_down() 的第1级调用时，最初的n仍然是它的初值1
  2. 函数每次调用都会返回一次，当函数执行完毕后，控制权将被传回上一级递归；程序必须按顺序逐级返回递归
  3. 虽然每级递归都有自己的变量，但是并没有拷贝函数的代码；程序按顺序执行函数中的代码，而递归调用就相当于有从头开始执行函数代码
  4. 递归函数必须包含能让递归调用停止的语句，因此每次形参都要使用不同的值
